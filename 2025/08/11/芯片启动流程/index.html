linux启动过程分析

发表于 2025-08-11 更新于 2025-12-03 分类于 启动流程
uboot启动流程

加载SPL

  • 片内ROM初始化最基本的硬件环境：关闭看门狗、初始化系统时钟、配置内存控制器、初始化一些外设控制器
  • 选择启动设备：根据芯片的启动引脚配置（Bootstrap Pin）来确定从哪个外部存储设备（如 NAND Flash、NOR Flash、eMMC、SD 卡等）读取后续的启动程序。
  • SPL 加载：该设备的特定位置读取少量代码到片内的 SRAM 中。

UBOOT 第一阶段

硬件设备初始化

  • 设置时钟系统（先使用外部时钟，待PLL稳定后，供给到系统作为系统时钟）
  • 关闭看门狗、关中断、启动ICACHE、关闭DCACHE和TLB
  • 关闭MMU

拷贝第二阶段代码

  • 准备内存空间，设置DDR（或SDRAM）地址（此时DDR已经可用了，但是没有MMU）
  • 初始化NANDflash相关接口和控制器
  • 通过nand接口从NAND拷贝数据到DDR（norf flash直接逐字节拷贝）
  • 将SDRAM的程序地址设置到PC，强制跳转

UBOOT第二阶段

  • 获取内存信息（地址、大小）
  • 初始化必要硬件设备（交互串口）
  • 将内核映像和文件系统（设备树）从FLASH拷贝到DDR中（根据情况选择是否需要解压）
  • 调用内核准备：CPU寄存器设置、进入超级模式、关中断、启动ICACHE、关闭MMU
  • 设置内核参数：控制台，第一个程序、内存起始地址、大小、根文件系统所在分区

多核CPU的启动

soc在启动阶段除了一些特殊情况外，一般都没有并行化需求。因此只需要一个cpu执行启动流程即可，这个cpu被称为primary cpu，而其它的cpu则统一被称为secondary cpu。

spin-table模式（ARMv7 及早期）

芯片上电后primary cpu开始执行启动流程，而secondary cpu则将自身设置为WFE睡眠状态，并且为内核准备了一块内存，用于填写secondary cpu的入口地址。

uboot负责将这块内存的地址写入devicetree中，当内核初始化完成，需要启动secondary cpu时，就将其内核入口地址写到那块内存中，然后唤醒cpu。

uboot启动时，若当前cpu为primary cpu，则继续执行启动流程。若当前cpu为slave cpu，则执行spin流程。执行wfe指令进入睡眠模式，以降低功耗

psci模式（ARMv8 及现代架构）

启动过程

环境准备

主 CPU（CPU0）在启动早期（head.S汇编阶段及start_kernel初始化过程中）会为从 CPU 准备必要的启动环境。

内存区域准备：

  • 从 CPU 的初始栈地址
  • 内核入口函数地址
  • 页表基地址（若已使能 MMU，需为从 CPU 配置临时映射）

硬件控制器初始化：

  • 初始化GIC（通用中断控制器）：确保从 CPU 能接收中断信号（如唤醒中断）。
  • 初始化SCU（对称多处理控制器，ARMv7） 或CCI（缓存一致性互连，ARMv8）：使能多核间的缓存一致性，避免内存访问冲突。

CPU复位
方式1：硬件复位（ARMv7 及早期）

  • 主 CPU 通过全局复位控制器寄存器（如CPU Reset Control Register）清除从 CPU 的复位信号（例如，写入特定值到RSTC寄存器的CPU1_RST位）。
  • 从 CPU 退出复位后，会从硬件固定的复位向量地址开始执行指令。

方式 2：PSCI 规范（ARMv8 及现代架构）

  • PSCI（Power State Coordination Interface） 是 ARM 定义的标准化电源管理接口，用于多核启动、休眠等操作。主 CPU 通过SMC（Secure Monitor Call） 指令调用 PSCI 固件（如 ATF 中的 BL31），发送PSCI_CPU_ON命令（参数为从 CPU 的 MPIDR 和启动地址）。
  • PSCI 固件负责释放从 CPU 的复位，并引导其跳转到指定的启动地址。

CPU执行启动代码
从 CPU 退出复位后，需先执行一段 “启动桩代码”（secondary_boot），这段代码通常由主 CPU 预先拷贝到从 CPU 的复位向量地址（或通过硬件映射确保可访问），主要功能包括：

  • 关闭局部中断：禁止从 CPU 在初始化完成前响应中断，避免干扰。
  • 设置初始栈：从主 CPU 准备的共享内存中读取栈地址，初始化SP（栈指针）。
  • 使能缓存与 MMU（可选）：若主 CPU 已配置好页表，从 CPU 会加载页表基地址（TTBR0）并使能 MMU，确保后续访问虚拟地址有效。
  • 同步多核状态：通过自旋锁或共享内存标志等待主 CPU 完成全局初始化（如cpu_online_mask的设置）。
  • 跳转到内核入口：最终跳转到secondary_start_kernel函数（Linux 内核为从 CPU 准备的初始化入口）。

调度器启动

  • 初始化自身的struct cpu结构体，注册到内核的 CPU 管理框架。
  • 使能本地中断控制器（GIC CPU interface），允许响应中断。
  • 加入全局 CPU 在线掩码（cpu_online_mask），标记为 “可用”。
  • 最终通过cpu_startup_entry进入 idle 进程，等待调度器分配任务，完成从 CPU 的启动。
